# 何志勇官方網站 — GitHub Pages 部署步驟

## 資料夾結構

```
github-pages/
├── index.html      ← 網站首頁
├── style.css       ← 樣式檔（已從 Weebly LESS 轉為標準 CSS）
├── custom.js       ← 互動功能（輪播、手風琴、複製帳號等）
└── images/         ← 所有圖片（52 張，已全部就位）
```

---

## 步驟一：建立 GitHub 帳號

1. 前往 https://github.com 註冊帳號（免費）
2. 建議使用英文帳號名稱，例如 `chihyungho` 或 `HoChihyung`
3. 完成 Email 驗證

---

## 步驟二：建立 Repository（程式碼倉庫）

1. 登入 GitHub 後，點選右上角 **+** → **New repository**
2. 填寫以下資訊：
   - **Repository name**：`chihyungho.github.io`
     （如果帳號名稱是 `chihyungho`，則網站網址會是 `https://chihyungho.github.io`）
   - **Description**（選填）：何志勇 2026 新竹市長參選人官方網站
   - 選擇 **Public**（公開）
   - **不要** 勾選 "Add a README file"
3. 點選 **Create repository**

---

## 步驟三：上傳檔案

### 方法 A：網頁上傳（最簡單）

1. 在剛建立的 Repository 頁面，點選 **uploading an existing file**
2. 將 `github-pages/` 資料夾裡的 **所有檔案與 images 資料夾** 拖曳到上傳區
   - `index.html`
   - `style.css`
   - `custom.js`
   - `images/` 資料夾（包含所有圖片）
3. 在 "Commit changes" 填寫 `初始上傳`
4. 點選 **Commit changes**

> **注意**：GitHub 網頁上傳一次最多 100 個檔案。images/ 資料夾有 52 張圖片，加上 3 個檔案共 55 個，一次可以全部上傳。

### 方法 B：使用 GitHub Desktop（推薦，方便日後更新）

1. 下載 GitHub Desktop：https://desktop.github.com
2. 安裝後登入你的 GitHub 帳號
3. 選擇 **Clone a repository** → 找到你剛建的 Repository → Clone
4. 將 `github-pages/` 資料夾裡的所有內容複製到 Clone 下來的資料夾
5. 回到 GitHub Desktop，你會看到所有變更
6. 填寫 Summary：`初始上傳`
7. 點選 **Commit to main** → **Push origin**

---

## 步驟四：啟用 GitHub Pages

1. 在 Repository 頁面，點選上方的 **Settings**（齒輪圖示）
2. 左側選單找到 **Pages**
3. 在 **Source** 區塊：
   - **Branch** 選擇 `main`
   - **Folder** 選擇 `/ (root)`
4. 點選 **Save**
5. 等待 1-2 分鐘，頁面會顯示：
   > Your site is live at `https://你的帳號名.github.io`

---

## 步驟五：設定自訂網域（保留 Weebly 原網址）

如果你想使用自己的網域名稱（例如 `chihyungho.tw` 或 `www.你的網域.tw`）：

### 5-1. 在 GitHub 設定自訂網域

1. 在 Repository **Settings** → **Pages** 頁面
2. 在 **Custom domain** 欄位輸入你的網域，例如 `www.chihyungho.tw`
3. 點選 **Save**
4. 勾選 **Enforce HTTPS**

### 5-2. 在網域 DNS 設定

到你的網域註冊商（例如 TWNIC、GoDaddy、Cloudflare）的 DNS 管理頁面，新增以下記錄：

**使用子網域（如 www.chihyungho.tw）：**

| 類型   | 名稱  | 值                          |
|--------|-------|-----------------------------|
| CNAME  | www   | 你的帳號名.github.io        |

**使用裸網域（如 chihyungho.tw）：**

| 類型 | 名稱 | 值              |
|------|------|-----------------|
| A    | @    | 185.199.108.153 |
| A    | @    | 185.199.109.153 |
| A    | @    | 185.199.110.153 |
| A    | @    | 185.199.111.153 |

> DNS 生效通常需要 10 分鐘到 48 小時。

### 5-3. 建立 CNAME 檔案

在 Repository 根目錄新增一個名為 `CNAME` 的檔案（無副檔名），內容只有一行：

```
www.chihyungho.tw
```

（替換成你實際使用的網域）

---

## 關於 Weebly 原網址

Weebly 提供的免費子網域（如 `xxxxx.weebly.com`）在 Weebly 關閉後將無法保留。

如果你原本使用的是 **自己購買的網域** 並指向 Weebly，只要將 DNS 改指向 GitHub Pages（步驟五），網址就能維持不變。

如果你使用的是 Weebly 免費子網域，建議儘早：
1. 註冊自己的網域（`.tw` 域名可在 TWNIC 或各域名商註冊）
2. 在所有文宣加上新網址

## 日後更新網站內容

### 更新文字或版面

1. 在 GitHub Repository 中找到 `index.html`
2. 點選鉛筆圖示（Edit this file）
3. 修改內容後，點選 **Commit changes**
4. 約 1 分鐘後網站自動更新

### 新增圖片

1. 進入 Repository 的 `images/` 資料夾
2. 點選 **Add file** → **Upload files**
3. 拖曳新圖片上傳
4. Commit changes

### 使用 GitHub Desktop 更新（推薦）

1. 在本機修改檔案
2. 打開 GitHub Desktop
3. 填寫更新說明 → Commit → Push

## 常見問題

**Q: 網站上傳後顯示空白？**
A: 確認檔案名稱是 `index.html`（全小寫），且位於 Repository 根目錄（不是在子資料夾裡）。

**Q: 圖片不顯示？**
A: 確認 `images/` 資料夾名稱正確，且圖片檔名大小寫與 HTML 中的一致。

**Q: GitHub Pages 有流量限制嗎？**
A: 免費方案每月有 100 GB 頻寬，對選舉網站來說綽綽有餘。

**Q: 網站是否支援 HTTPS？**
A: 是，GitHub Pages 自動提供免費 HTTPS 憑證。
